Key points

  • Preprocessors include statement 
    enum __EXEN_ENUM_NAME
{
  #define __EXEN_ENUM_ENTRY(__enum_entry) __enum_entry,
  #include __EXEN_INCLUDE_ENUM(__EXEN_ENUM_NAME)
  #undef __EXEN_ENUM_ENTRY
};

    It’s possible to include a file at any position of the code

  • Preprocessors include-filename as 'define'-statement 
    ...

  #include __EXEN_INCLUDE_ENUM(__EXEN_ENUM_NAME)

...

    The preprocessing-step can be divided into 2 steps, first the define-macros will be substituted second the include statements will be resolved.

    This allows the filename to be set as a define-macro.

More key points

  • How to get the entry-count without polluting the enum 
    namespace detail {
#define __EXEN_ENUM_P __EXEN_ENUM_P_DECL(__EXEN_ENUM_NAME)
enum class __EXEN_ENUM_P
{
  #define __EXEN_ENUM_ENTRY(__enum_entry) __enum_entry,
  #include __EXEN_INCLUDE_ENUM(__EXEN_ENUM_NAME)
  #undef __EXEN_ENUM_ENTRY
  COUNT // <--
};
} // namespace detail

    There are two enumerations, one is hidden in a namespace called detail which contains the polluting last entry COUNT.

Magic key points

  • How to get for the enumeration the right underlying type 
    constexpr std::uint64_t count() { return static_cast<std::uint64_t>(detail::__EXEN_ENUM_P::COUNT); }

using optimal_unsigned_integer_size_type =
  std::conditional<std::numeric_limits<std::uint8_t>::max() >= count(),
    std::uint8_t,
    std::conditional<std::numeric_limits<std::uint16_t>::max() >= count(),
    std::uint16_t,
      std::conditional<std::numeric_limits<std::uint32_t>::max() >= count(),
      std::uint32_t,
      std::uint64_t
      >::type
    >::type
  >::type;

    With some C++ template metaprogramming MAGIC

Even more key points

  • To string converter 
    namespace detail {
constexpr std::array<const char*, count()> enum_names =
{{
  #define __EXEN_ENUM_ENTRY(__enum_entry) #__enum_entry,
  #include __EXEN_INCLUDE_ENUM(__EXEN_ENUM_NAME)
  #undef __EXEN_ENUM_ENTRY
}};
} // namespace detail

/** Returns the string representation of the given enumeration value. */
constexpr const char* name(__EXEN_ENUM_NAME value) {
  return detail::enum_names[static_cast<std::uint64_t>(value)];
}

    An Array of Strings is created in the hidden detail namespace. This array contains at the position of a corresponding enum-entry the string representation of it.

    So with the function called name( enum_type ) it’s possible to return the string of the corresponding enumeration entry.

  • Stream operator overloading 
    std::ostream& operator<<(std::ostream& stream, enum_type rhs)
{
  stream << std::string(enum_type::name(rhs));
  return stream;
}

    to top it all off.

    So with this, the enumeration-entry is automatically deduced in a stream statement and will therefore return a string instead of the number itself.
